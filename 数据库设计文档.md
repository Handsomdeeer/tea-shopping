# 1. 数据库设计概述

## 1.1 设计原则
- 遵循第三范式（3NF），减少数据冗余
- 所有表统一使用 `id` 作为主键，类型为 BIGINT
- 每张表包含 `create_time` 和 `update_time` 字段
- 逻辑外键约束，在应用层维护关系
- 字符集统一使用 utf8mb4

## 1.2 命名规范
- **表名**：小写字母 + 下划线，复数形式（如：users, products）
- **字段名**：小写字母 + 下划线（如：user_name, create_time）
- **索引名**：idx_字段名（普通索引），uk_字段名（唯一索引）
- **主键**：统一命名为 id
