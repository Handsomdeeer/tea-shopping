# 软件需求规格说明

## （Software Requirement Specification）

# 1 引言

## 1.1 目的

本文档描述了一个潮茶商城系统 ChaoCha E-Commerce V1.0 版本的软件功能需求和非功能需求，其阅读对象是本项目的客户、开发和维护系统的开发团队成员。

## 1.2 文档约定

本文档的命名遵从如下规范：

**SRS-XXX-YYY**：需求标识  
XXX 表示需求类型。需求类型分为 3 类：接口需求 INT、功能需求 FUN、非功能需求 NTF；  
YYY 表示具体需求项，用 3 位数字表示。

**UC-XXX**：用例标识  
XXX 表示具体用例项，用 3 位数字表示。

**ANL-DGM-UCR-XXX**：用例实现交互图标识  
XXX 表示具体用例实现交互图项，用 3 位数字表示。

**ANL-XXX-CLS-YYY**：分析类标识  
XXX 表示分析类类型。分析类类型分为 3 类：边界类 BOD、控制类 CTR、实体类 ENT；  
YYY 表示具体分析类项，用 3 位数字表示。

## 1.3 预期的读者和阅读建议

- 项目管理人员可以根据功能的优先级来安排项目的开发进程；
- 项目开发人员可以根据分析模型来指导系统设计和详细设计；
- 测试人员可以根据详细的用例描述来指导测试用例的开发。

## 1.4 产品的范围

潮茶商城系统 ChaoCha E-Commerce 是一个基于 WEB 的应用软件，它专注于茶叶及相关产品的在线销售。系统支持用户在线浏览商品、管理购物车、下单购买、查看订单状态等电商核心功能。同时，管理员使用计算机实现对商品、订单、用户等的全面管理。

# 2 综合描述

## 2.1 产品的前景

ChaoCha E-Commerce 系统是一个应用计算机的新系统，它取代了传统线下茶叶销售模式，通过互联网为用户提供便捷的在线购物体验，提高茶叶销售的业务效率。

该系统有注册用户和管理员两种用户，普通用户必须首先进行注册才可以使用该系统。图书管理员负责添加、更新和删除系统中的商品信息，并处理订单。注册用户可以按照分类或关键词检索商品信息，并且可以将商品添加到购物车进行购买。

该系统应该是可定制的，管理员可以设置商品管理规则；该系统的设计应该支持扩展性，希望可以演化成若干版本，最终实现对各种商品的完整管理，并能够适用于更大范围的电商平台。

## 2.3 用户类和特征

| **用户类**  | **说明**                                                                                                                                                                                           |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **注册用户** | 注册用户是购买茶叶的消费者，他们希望能够在线浏览商品、管理购物车、下单购买和查看订单。预计系统上线初期有 5000 注册用户，其中估计有 2000 人预计平均每星期每人使用 ChaoCha E-Commerce 系统 3 次（来源：根据电商平台的用户使用数据）。所有的注册用户都可以从办公室、家庭等访问互联网，用户希望系统能够提供便捷的购物流程和订单跟踪服务，并能够收到订单状态通知。 |
| **管理员**  | 目前有 3 名管理员，他们负责添加、更新和删除系统中的商品信息，处理订单（发货、退款审核等），管理用户信息和查看统计报表。管理员需要接受培训，学会如何使用计算机、WEB 浏览器和 ChaoCha E-Commerce 系统。                                                                                 |

## 2.4 运行环境

- 系统应该运行在 Windows 7 以及以上版本的平台上；
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）的最新版本；
- 服务器采用 Linux 或 Windows Server 操作系统；
- JDK 版本：Java 17；
- 数据库：MySQL 8.0+。

## 2.5 设计和实现上的限制

- 系统的开发文档将遵循公司的开发标准 V1.0；
- 软件开发语言为 Java，框架采用 Spring Boot 4.0.4；
- 数据库系统为 MySQL 8.0+；
- 前端采用 Thymeleaf 模板引擎 + HTML5 + CSS3 + JavaScript。

## 2.6 假设和依赖

假设使用该文档的人具有 Web 界面背景知识和一定的图形分析模型概念（主要包括用例图和时序图）。

假设用户在执行了任何一条功能后，都可以终止进一步的操作。

本系统依赖第三方支付平台（微信支付、支付宝）提供支付二维码生成和回调接口。

# 3 外部接口需求

## 3.1 用户界面

SRS-INT-001：系统对所显示的每个 HTML 页面都提供帮助链接，说明该页面的操作。

SRS-INT-002：WEB 页面的操作除了综合使用鼠标和键盘共同完成之外，还可以只通过键盘来完成。

SRS-INT-003：用户界面必须符合《Web 界面规范》，采用茶文化主题色（青绿色系）。

SRS-INT-004：详细的界面需求可参考《用户界面规格说明（User Interface Specification）》。

## 3.2 硬件接口

无。

## 3.3 软件接口

SRS-INT-005：与现有的邮件系统连接给用户发送订单状态通知邮件。

SRS-INT-006：与短信服务系统连接给用户发送验证码和订单通知短信。

SRS-INT-007：与第三方支付平台（微信支付、支付宝）接口连接，用于生成支付二维码和接收支付回调。

## 3.4 通信接口

SRS-INT-008：系统采用 HTTP/1.1 或 HTTPS 协议进行通信。

SRS-INT-009：数据格式采用 JSON 格式，字符编码为 UTF-8。

# 5 非功能需求

## 5.1 性能需求

SRS-NTF-001：在每天的上午 10:00－12:00 和下午 2:00－4:00 是高峰时间，系统将能适应 1000 个用户同时使用，平均每个会话估计持续 15 分钟。

SRS-NTF-002：系统的所有 WEB 页面在百兆的光纤网上全部显示时间的最大值是 3 秒钟。

SRS-NTF-003：系统对用户请求的最大响应时间是 2 秒钟，在此时间内将响应结果显示在屏幕上。

上述性能需求要求的软硬件环境如下：

- Intel CPU 2GHz 以上；
- 内存容量 4GB 以上；
- SSD 硬盘，容量 100GB 以上；
- 操作系统 Windows 10 或 Linux；
- JDK 17 运行环境。

## 5.2 安全设施需求

无。

## 5.3 安全性需求

SRS-NTF-004：用户必须使用用户名和口令登录系统成功后才可以执行系统其他操作。

SRS-NTF-005：系统只允许用户查看自己的订单和购物车信息，而不能查看别人的记录。

SRS-NTF-006：不能对系统中其他运行的软件造成破坏，或造成操作系统死机。

SRS-NTF-007：用户密码必须加密存储（采用 BCrypt 算法）。

SRS-NTF-008：敏感信息传输必须采用 HTTPS 加密。

## 5.4 软件质量属性

SRS-NTF-009：为了便于代码的维护，编码必须遵从《Java 语言编程规范》。

SRS-NTF-010：ChaoCha E-Commerce 系统将在每周 7 天每天 24 小时的 99% 时间可用。

SRS-NTF-011：如果在下单、支付等操作执行完成之前，用户和系统的连接中断，ChaoCha E-Commerce 系统将恢复到操作执行之前的状态。

SRS-NTF-012：系统应该具有良好的可扩展性，支持集群部署和负载均衡。

## 5.5 业务规则

SRS-NTF-013：订单创建后 30 分钟内未完成支付，系统将自动取消订单。

SRS-NTF-014：用户确认收货后 7 天内可以发起退货申请。

## 5.6 用户文档

SRS-NTF-015：系统将提供在线联机帮助系统。

SRS-NTF-016：系统将为新用户提供操作指南和常见问题解答。

# 6 其他需求

SRS-NTF-017：系统能够在不同的操作系统平台（Windows、Linux）上运行。

SRS-NTF-018：提供安装部署文档和运维手册。

SRS-NTF-019：对于一个没有经验的用户而言，经过一个小时的培训就可以使用系统的所有基本功能。

SRS-NTF-020：系统应该支持主流的浏览器（Chrome、Firefox、Safari、Edge）。
